## 11. タグ運用ルール

### 必須ルール

1. **層タグ 1 件は必須**（`type` と同じ値）
2. **ドメインタグ最低 1 件**（#障害福祉 + 下位）
3. **過剰タグ禁止**（1 ノート 10 件以内推奨）
4. **重複語彙を避ける**（例: `#総合支援法` と `#障害者総合支援法` を混在させない）

### 推奨

- 改正対応時は改正タグを追加（`#令和6改正` 等）
- 新設ページには `#新設`
- 複数分野に跨る場合は複数タグ付与

### 禁止

- **個人名タグ**（仮名含む）
- **特定事業者タグ**
- **地域固有名詞**（都道府県名以外）

---

## 12. 新規タグ追加手順

1. 既存タグで表現できないことを確認
2. 本ファイルの適切なセクションに追記
3. コミットメッセージ例: `refactor(meta): taxonomy に #新タグ名 を追加`
4. 既存ノートへの遡及付与は任意

## 13. data-wiki/CLAUDE.md §5 との対応

data-wiki の標準タグのうち、welfare-graph でもそのまま使用するもの:

| data-wiki タグ | welfare-graph 扱い |
|---|---|
| `#障害福祉` | 採用 |
| `#生活保護` | 採用（低使用頻度） |
| `#計画相談` | 採用 |
| `#不服申立て` | 採用 |
| `#権利擁護` | 採用 |
| `#意思決定支援` | 採用 |
| `#虐待対応` | 採用 |
| `#報酬改定` | 採用 |
| `#法改正` | 採用（`#令和X改正` と併用可） |
| `#判例研究` | 採用 |

追加カテゴリ（welfare-graph 固有）:

- 障害種別系（#知的障害 / #発達障害 等）
- 改正対応系（#pending-amendment / #archived 等）
- エビデンス系（#エビデンス高 等）
