# ニーズ整理票

## 概要

本人・家族の **「願い・困りごと」** を **「支援のニーズ」** に翻訳・構造化するシート。計画相談支援の中核ツール。

「本人は何がしたいか／何が困るか」→「どんな支援があればそれが実現するか」の論理展開を可視化する。

## 標準的な構成（サポートブック準拠）

| 欄 | 内容 | 情報源 |
|---|---|---|
| **本人の希望（夢）** | 本人の言葉そのもの | 本人面接 |
| **家族の希望** | 家族の願い | 家族面接 |
| **支援者としての見立て** | 専門職の見方 | 支援者合議 |
| **解決すべき課題** | 優先順位付け | 三者統合 |
| **支援の方向性** | 基本方針 | 協議 |
| **具体的な支援内容** | サービス・資源 | 計画作成 |

## 3つの視点の統合

### ① 本人視点
- 本人の **言葉そのもの** を残す
- 「分からない」「考えたことない」も記録
- 意思表明困難な場合は行動・反応から推測

### ② 家族視点
- 家族の願い・不安
- 介護負担・将来不安
- 家族間の意見相違も記録

### ③ 支援者視点
- 医学的・福祉的見立て
- 法令・ガイドラインから見た必要性
- 社会資源の現実

### 3視点の相違の扱い

よくあるケース:

- **本人「働きたい」/ 家族「無理」/ 支援者「就労A型候補」**
- **本人「GH嫌」/ 家族「親亡き後心配」/ 支援者「段階的移行検討」**
- **本人「恋愛したい」/ 家族「問題児にしないで」/ 支援者「権利として尊重」**

→ [[64_Methods/意思決定支援会議]] で **本人意思を優先しつつ合意形成**。

## ICF 観点での整理

[[62_Frameworks/ICF]] の6領域でニーズを分類すると漏れがない:

| 領域 | チェックポイント |
|---|---|
| 心身機能 | 健康・医療的ケア |
| 活動 | 日常生活動作・セルフケア |
| 参加 | 就労・地域活動・余暇 |
| 環境因子 | 家族・住居・制度 |
| 個人因子 | 年齢・性格・価値観・強み |

## 「困りごと」から「ニーズ」への変換

### 例1
- **困りごと**: 「朝起きられない」
- **ニーズ**: 「規則正しい生活リズムの確立」
- **支援**: 生活訓練・訪問介護・服薬調整

### 例2
- **困りごと**: 「パニックになる」
- **ニーズ**: 「見通しが立つ環境と代替行動の獲得」
- **支援**: [[64_Methods/構造化支援]]・[[64_Methods/ABC分析]]・[[64_Methods/強度行動障害支援]]

### 例3
- **困りごと**: 「親が高齢で将来不安」
- **ニーズ**: 「段階的な自立生活の準備」
- **支援**: [[66_Services/短期入所]] 定期利用・GH体験利用・[[00_MOC/Resilience]]

## 優先順位付け

### 緊急度×重要度マトリクス

| | 重要 高 | 重要 低 |
|---|---|---|
| **緊急 高** | 即対応 | 段階的対応 |
| **緊急 低** | 計画的対応 | 観察継続 |

### 「緊急 高 × 重要 高」の例

- 虐待・被害の疑い → [[60_Laws/障害者虐待防止法]] 通報
- 自傷・他害の顕在化 → 医療連携・[[64_Methods/強度行動障害支援]]
- 家族の緊急介護困難 → [[66_Services/短期入所]] 緊急利用

## 書式とツール

### 厚労省サポートブック準拠

標準様式がサポートブックに掲載。自治体によりカスタマイズ。

### デジタル化

- Excel/Google Sheets
- 専用ソフト
- Obsidian ノート（本vault）

### 本vault での運用

`10_People/P-XXXX.md` の中に統合:
- 本人の希望 → `## 本人の願い` セクション
- 家族の希望 → `## 家族の意向`
- 支援者見立て → `## 支援方針` / `## 推奨ケア`
- 課題・計画 → `## 支援方針（{年度}）` / `## 根拠付き支援仮説`
- `relations` で関連ページへリンク

## 計画相談でのワークフロー

### 1. 初回面接（1-2回）

- 本人の願いを聞く
- 家族の意向を聞く
- エコマップ作成
- 医学的・福祉的情報の収集

### 2. ニーズ整理

- 3視点の統合
- 優先順位付け
- ICF で網羅チェック

### 3. 計画案作成

- [[66_Services/計画相談支援]] のサービス等利用計画
- ニーズ→目標→サービスの論理展開

### 4. サービス担当者会議

- 関係者と計画案を協議
- 合意形成

### 5. 計画確定・モニタリング

- 実施状況確認
- ニーズ変化の把握
- 計画見直し

## 注意点

### 本人の願望が「社会通念」と衝突した時

- 恋愛・結婚・出産
- 危険な趣味・活動
- 高額な消費

→ 安全確保しつつ、**権利として尊重** する方向を模索（[[62_Frameworks/パーソンセンタードプランニング]]）。

### 「表現できない願望」

- 重度知的・意思表明困難
- 過去の経験から「願うこと」を諦めている
- 家族の意向に塗りつぶされている

→ 時間をかけて聞き取り、過去の言動・行動から推測、アドボケイト活用（[[61_Guidelines/意思決定支援ガイドライン]]）。
