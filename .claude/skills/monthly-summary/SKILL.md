# monthly-summary skill

## 用途
相談支援専門員が毎月のモニタリング報告書を作成する際、vault 内のエピソード群から定型的なサマリを抽出する。

## 入力
- 当事者ID（必須）: 例 `P-0001`
- 対象月（必須）: `YYYY-MM` 形式（例 `2026-03`）
- 出力形式（任意）: `report`（報告書風・デフォルト） / `table`（表形式のみ）

## 実行手順（Claude 向け）

1. **人物ノート取得**
   - `10_People/{id}.md` を Read し基本情報取得

2. **対象月のエピソード収集**
   - `20_Episodes/` から `{YYYY-MM}` と `{id}` を含むファイルを Glob
   - 全件 Read し、日付順に並べる

3. **メトリクス算出**
   - 件数、severity 別件数、mood 分布
   - 前月比（可能なら前月も取得して比較）

4. **パターン抽出**
   - 類似キーワード（気づき欄の名詞）をカウント
   - 3回以上出現したら「知見化候補」として注記

5. **CareRole 進捗**
   - `50_Resilience/CareRoles/` で `updated` が該当月のものを抽出
   - substitute_status の変化を記録

6. **レポート生成**
   - 上記フォーマットで出力
   - モニタリング報告書への転記を意識した見出し構成

7. **保存**
   - デフォルト保存先: `20_Episodes/monthly/{YYYY-MM}_{id}_summary.md`
   - ファイル先頭に frontmatter を付ける:
     ```yaml
     ---
     type: monthly_summary
     person: P-XXXX
     month: YYYY-MM
     generated: YYYY-MM-DD
     ---
     ```

## 注意事項
- 実名は一切含めない
- エピソードがゼロ件の場合は「該当月記録なし」と明示し、理由（訪問未実施/体調等）を確認するよう促す
- severity: critical が含まれる月は、冒頭に⚠️マークで強調
