# realname-check skill

## 用途
- hook は PreToolUse で Write/Edit 時にのみ動作する
- 既存ノートへの実名混入を**定期監査**するためのバッチスキャン
- 外部から markdown をインポートした後のクリーンアップ

## 入力
- スコープ（任意）: ディレクトリパス。デフォルトは vault 全体（`00_MOC`, `10_People`, `20_Episodes`, `30_Insights`, `40_Stakeholders`, `50_Resilience`）
- 除外パス（任意）: デフォルトで `90_Meta/alias_map.md` と `90_Meta/templates/` を除外

## 検出ルール
Hook と同じパターンセットを使用:
1. 漢字フルネーム（空白区切り）: `[漢字]{2,4} [漢字]{2,4}`
2. カタカナフルネーム: `[カタカナ]{2,8} [カタカナ]{2,8}`
3. 電話番号: `0\d{1,4}-\d{1,4}-\d{3,4}`
4. 郵便番号: `\d{3}-\d{4}`

例外:
- サンプル名「山田 太郎」「山田 花子」はホワイトリスト
- `<!-- allow-realname -->` を含むファイルは除外

## 実行手順（Claude 向け）

1. **対象ファイル列挙**
   - Glob で `.md` を列挙（除外パスをフィルタ）

2. **Grep で一次検出**
   - Grep ツールで各パターンを順に実行
   - 結果を集約

3. **レポート生成**
   ```markdown
   # 実名検出レポート
   
   **実行日時**: YYYY-MM-DD HH:MM
   **検査範囲**: vault全体
   **検査ファイル数**: N
   
   ## 検出結果（件数: M）
   
   ### [漢字フルネーム]
   - `ファイルパス:行番号` -- '検出文字列'
   
   ### [カタカナフルネーム]
   - ...
   
   ### [電話番号]
   - ...
   
   ## 推奨アクション
   1. 検出ファイルを開き、実名を仮名IDに置換
   2. 実名は `90_Meta/alias_map.md` に登録
   3. 再実行してクリーン状態を確認
   ```

4. **保存（任意）**
   - `90_Meta/audit/realname_check_{YYYY-MM-DD}.md` に保存

## 注意事項
- このスキル自体が実名を含むレポートを生成する可能性があるため、出力は**検出行の周辺のみ**に絞る
- 一致文字列は短縮表示（最初の数文字＋伏字）にする選択肢も提供
- 誤検出（地名・作品名・一般名詞）が多い場合は、ホワイトリスト拡充を提案
