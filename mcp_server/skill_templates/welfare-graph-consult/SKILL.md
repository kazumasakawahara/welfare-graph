# welfare-graph-consult skill

## いつ使うか

ユーザーが以下のような質問をしたとき:

- 「強度行動障害のある人にどう関わればよい？」
- 「成年後見制度を利用するには？」
- 「就労継続支援B型と就労移行支援の違いは？」
- 「親が高齢になってきたが、どんな備えが必要？」
- 「虐待の疑いがあるが、通報義務はどうなっている？」
- 「30 代で知的障害のある女性の支援、どんな選択肢がある？」
- 「自閉症の方への性教育、何から始める？」
- 障害福祉サービス・障害者総合支援法・意思決定支援等に関する一般的な相談

**判定基準**: 障害福祉領域のキーワード（障害・知的・発達・精神・自閉・サービス・支援区分・相談支援・意思決定・後見・虐待・通報・等）が含まれていれば、まず本スキルを起動する。

## 何を絶対に守るか

### 必須ルール

1. **一次情報を必ず引用する**: welfare-graph MCP の tool 呼び出し結果から `source_url` を取得し回答に含める
2. **改正動向を必ず確認する**: `check_amendment_status` を併走実行し、`pending-amendment` や `expired` があれば「最新確認が必要」と明示
3. **免責事項を必ず添える**:
   - 「個別ケースの判断は担当の相談支援専門員・医師・弁護士の合議で行ってください」
   - 「本人の意思決定支援を最優先とします（意思決定支援ガイドライン参照）」
   - 「本回答は学習・参考目的であり、法的・医学的判断ではありません」
4. **実名・特定可能情報を返さない**: MCP は仮名情報のみ返すが、ユーザーが実名を出した場合も会話で応答に含めない
5. **不足情報を能動的に指摘する**: 質問だけでは判断できない要素（例: 区分・年齢・家族構成）を回答末尾で「追加で必要な情報」として列挙

### 禁止事項

- 一次ソース URL なしで法令・サービスの内容を断定的に述べる
- welfare-graph MCP を一切呼ばずに「障害福祉の一般論」だけで答える
- 利用者個人を特定できる情報を回答に含める
- 「絶対」「必ずこうすべき」等の断定的表現を本人意思を超えて行う

## 推奨される実行手順

### 手順 1: 質問の分類と起点選定

ユーザーの質問を以下の 5 タイプに分類:

| タイプ | 起点 tool |
|---|---|
| A. 法的義務・通報・コンプライアンス | `find_applicable_laws(situation)` |
| B. 障害特性 → 支援方法 | `find_methods_for_disorder(disorder_name)` |
| C. サービス検索 | `find_services_for_profile(disorders, keywords)` |
| D. 総合相談（特定ケースの支援方針） | `support_hypothesis(profile_text)` |
| E. 概念・用語の意味 | `search_vault(query)` → `get_note(nid)` |

### 手順 2: コア tool を呼ぶ

選んだ tool で初期検索。結果が乏しい場合は `search_vault` でフォールバック。

### 手順 3: 詳細を取得

主要該当ノートについて `get_note(nid)` で全文取得し、根拠として引用。

### 手順 4: 関連知識を辿る

特に必要があれば `get_related(nid, depth=2)` で関連法令・サービス・技法を補完。

### 手順 5: 改正動向を確認

`check_amendment_status` を呼び、関連分野の `pending-amendment` を確認。

### 手順 6: 回答を構造化して返す

```
## 概要回答
（1-2 段落）

## 根拠（一次情報）
- [法令/ガイドライン名]（出典: URL）
- 関連条項・記載箇所

## 該当する支援技法・サービス
- ...

## ⚠ 注意・改正動向
- （pending-amendment があれば）

## 追加で確認すべき情報
- （ユーザーから取得が必要な情報）

## 免責
- 個別判断は担当の相談支援専門員等の合議で行ってください
- 最新情報は e-Gov・厚生労働省サイトでご確認ください
```

## 例 1: 「強度行動障害のある利用者への支援はどうすべき？」

### 実行内容

```
1. find_methods_for_disorder('強度行動障害')
2. search_vault('強度行動障害', layer='61_Guidelines')
3. get_note('61_Guidelines/強度行動障害支援者養成研修テキスト')
4. check_amendment_status('66_Services')   # 報酬改定の確認
```

### 期待される回答骨格

```markdown
## 概要回答
強度行動障害は専門研修を受けた職員による構造化された支援が基本です。
厚生労働省の研修テキスト（基礎・実践）が標準的な指針となります。

## 根拠
- 強度行動障害支援者養成研修テキスト（厚生労働省）
  https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/0000063285.html
- 障害者総合支援法（行動援護・重度訪問介護の根拠法）

## 該当する支援技法
- 構造化支援（TEACCH 由来）
- 応用行動分析（ABA）
- PECS（コミュニケーション支援）

## サービス選択肢
- 行動援護（区分 5 + 行動関連 10 点以上）
- 重度訪問介護（区分 4 以上）
- 共同生活援助（強度行動障害支援加算）

## ⚠ 注意
- 2027 年度報酬改定で強度行動障害関連加算の見直し議論あり（要追跡）
- 物理的拘束や薬物のみによる対応は身体拘束適正化の手引違反

## 追加で必要な情報
- 区分認定の有無
- 現在利用中のサービス
- 家族支援の状況
- 主な行動の具体（自傷/他害/睡眠/食/排泄/こだわり）

## 免責
- 個別ケースの最終判断は担当の相談支援専門員・主治医・関係機関の合議で行ってください
- 本回答は学習・参考目的のものです
```

## 例 2: 「親が高齢になってきた。何から準備すべき？」

```
1. search_vault('親亡き後', limit=10)
2. find_applicable_laws('親が高齢で将来的に介護できなくなる場合の備え')
3. find_services_for_profile(keywords=['共同生活援助', '生活介護', '成年後見'])
4. get_note('60_Laws/成年後見制度')
```

## エラー対応

| 状況 | 対応 |
|---|---|
| MCP サーバーが未起動 | 「MCP 接続不可。Claude Desktop / Code 設定を確認してください」と案内 |
| 該当ノートなし | 「welfare-graph には該当情報なし。直接 e-Gov 等で確認推奨」と案内 |
| 検索ヒット 0 件 | キーワードを変えて再検索、または近接概念で再検索 |

## 設定確認

このスキルは以下の MCP サーバーが利用可能であることを前提とします:

```json
{
  "mcpServers": {
    "welfare-graph": {
      "command": "welfare-graph-mcp",
      "env": {
        "WELFARE_GRAPH_VAULT": "/path/to/welfare-graph"
      }
    }
  }
}
```

未設定なら、ユーザーに [`docs/MCP_SETUP.md`](https://github.com/kazumasakawahara/welfare-graph/blob/main/docs/MCP_SETUP.md) を案内する。
