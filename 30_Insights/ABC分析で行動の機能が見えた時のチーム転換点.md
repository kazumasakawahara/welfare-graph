# ABC分析で行動の機能が見えた時のチーム転換点

## 要旨

ABC分析を **2週間以上蓄積** すると、チームの **行動の見方が「病理」から「コミュニケーション」へ転換** する。この転換点を経ると、その後の支援計画の質が飛躍的に向上する。

## 典型的な経過

### フェーズ1: 記録開始（1-2週間）
- 支援員が「暴れた」「大声」等の **主観的記録** を続ける
- フォーマットが不揃い
- チーム会議では「困った」の共有に終始

### フェーズ2: 客観化（2-4週間）
- A（先行）・B（具体的行動）・C（結果）の3項に絞った記録へ
- 「暴れた」→「腕を掴んだ後に床に座り込んだ（5分）」のような具体化
- 時刻・場所・関わった人を記録

### フェーズ3: パターン発見（4-6週間）
- 特定の A（先行事象）で B（行動）が起きる **反復** に気付く
- 例: 昼食の匂いが強い日に他害発生
- 例: 特定職員の休みの日に自傷増加
- 「偶然ではない」とチーム全体が認識

### フェーズ4: 機能仮説（6-8週間）
- 「要求獲得 / 回避 / 注目獲得 / 感覚刺激」 のどれか
- 本人が **その行動で何を得ているか** が言語化される
- 「わがまま」→「コミュニケーション手段」への見方転換

### フェーズ5: 代替行動の設計
- 同じ機能を果たす **適切な代替行動** を教える
- 例: 嫌な課題から逃げたい → 「休憩」カードを渡す練習
- 例: 注目を得たい → 定期的な声かけルーティン

## 典型的な「気付き」の例

### ケース1
- **仮説前**: 「週末のイライラで月曜に他害」
- **仮説後（ABC分析で判明）**: 「日曜夜の家族面会で興奮 → 翌朝の早い起床困難 → 睡眠不足 → 感情調整困難」
- **介入**: 面会時間の短縮 + 月曜朝のルーティン調整 → 月曜の他害ほぼ消失

### ケース2
- **仮説前**: 「自傷は本人のストレス発散」
- **仮説後**: 「自傷すると職員が駆けつける = 注目獲得機能」
- **介入**: 予防的な声かけ（5分毎の確認）で注目機会を定期化 → 自傷減少

### ケース3
- **仮説前**: 「食事中の異食行動はこだわり」
- **仮説後**: 「特定の食感が苦手 → 食事を早く終わらせたい（回避機能）」
- **介入**: メニュー調整 + 完食圧力の解除 → 異食消失

## チーム転換のサイン

転換点を超えたチームは次の特徴を示す:

- [ ] 「本人が悪い」ではなく「環境が合っていない」と考える
- [ ] 行動を止めるのではなく、機能を満たす別の方法を一緒に考える
- [ ] 記録が **観察可能な事実** 中心になる
- [ ] ケース会議が **仮説検証** の議論になる
- [ ] 支援員間で **同じ言語** で本人を語れるようになる

## 失敗パターン

### 記録が続かない
- フォーマットが複雑すぎる
- 記録の価値が共有されていない
- 記録者が偏る（特定の人だけ）

**対策**: シンプルな1行記録から、スマホ音声入力、チーム内で記録ローテーション

### 仮説生成で止まる
- 「機能仮説」が立てられても介入設計に至らない
- スーパービジョン不足

**対策**: [[67_Orgs/基幹相談支援センター]] ・[[67_Orgs/発達障害者支援センター]] のコンサル活用

### 短期で効果を求めすぎる
- 2週間で効果を期待して挫折
- ABCは **最低2-3か月** のスパンで見る

**対策**: 「今月はパターン発見、来月は仮説、再来月は介入検証」のフェーズ合意

## 計画相談での使いどころ

- 新規ケースで行動上の困難がある時、**事業所に ABC記録の導入を提案**
- モニタリング時に「最近の ABC記録から見える傾向」を情報収集
- 事業所の質評価の一指標として ABC分析の体系的実施を確認
