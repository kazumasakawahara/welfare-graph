# 配布告知テンプレート集

相談支援専門員コミュニティへ welfare-graph を紹介する際のテンプレート。媒体ごとに分かれています。

## 1. メーリングリスト・メール用（長文版）

```
件名: 【無料・オープンソース】相談支援専門員のための知識グラフ「welfare-graph」公開

相談支援専門員の皆さま

日々の業務でこのような困りごとはないでしょうか:

- 法令・ガイドライン・サービスの情報が散在していて参照しづらい
- 利用者ごとの記録と知識を結びつけて整理したい
- 改正情報を追いきれず、古い情報のまま運用している不安
- 性教育・権利擁護など重要だが扱いづらいテーマの情報整理が難しい

このたび、上記の課題に応える **無料・オープンソースの知識グラフツール**
「welfare-graph」を公開しました。

【概要】
- Obsidian（無料アプリ）上で動く障害福祉知識ノート集（120 ページ以上）
- 9 層構造（法令・ガイドライン・理論・障害特性・支援技法・アセスメント・サービス・機関・利用者）
- 利用者ノートと知識ノートを重み付きリンクで接続
- 仮名 ID と暗号化で個人情報保護を仕組み化
- 法令改正に追随する「進化する知識グラフ」機構を内蔵
- 完全無料・商用利用可（CC BY-SA 4.0 / MIT デュアルライセンス）

【特徴】
- Obsidian 初心者・プログラミング未経験者でも使える詳細な手順書つき
- 性教育・成年後見・親亡き後設計など、タブー視されがちなテーマも収録
- AI（Claude）と連携すると支援仮説の自動生成も可能（任意）
- Neo4j と連携すると地域分析も可能（任意）

【入手】
- リポジトリ: https://github.com/kazumasakawahara/welfare-graph
- 使用説明書: https://github.com/kazumasakawahara/welfare-graph/blob/main/docs/USER_GUIDE.md
- クイックスタート: https://github.com/kazumasakawahara/welfare-graph/blob/main/docs/QUICKSTART.md

【コミュニティ協働】
法改正・ガイドライン更新があった際の追従にご協力いただけると幸いです。
GitHub の Issue / Pull Request で寄付できます（CONTRIBUTING.md 参照）。

【免責】
本ツールは学習・参考・実務補助を目的とします。個別ケースの法的・医学的判断は、
担当相談支援専門員・医師・弁護士等の合議で行ってください。

質問・改善提案は GitHub Issues までお願いします。

【作者】
川原 一誠（仮名・所属協会名等を実運用時に記載）

----
welfare-graph プロジェクト
https://github.com/kazumasakawahara/welfare-graph
```

## 3. 学会発表・研究会発表用（要旨）

```
題目: 相談支援専門員のためのオープン知識グラフ "welfare-graph" の開発と公開

背景:
障害福祉領域では法令・ガイドライン・サービス情報が膨大かつ改正頻度が高く、
個別の相談支援専門員が最新情報を維持することは困難である。

目的:
オープンソースで継続的に進化する知識グラフを構築し、相談支援専門員コミュニティ
の集合知として共有資産化する。

方法:
- Obsidian を基盤とした 9 層構造の知識ノート（法令〜利用者）
- 重み付きリンク（type + weight）で関係を意味づけ
- LLM-Wiki 方式で人間可読 + 機械処理の両立
- archived/ と amendment_history による版管理
- GitHub Actions で月次自動健全性チェック
- Claude Code スキルで支援仮説生成・改正取込を自動化

結果:
2026 年 4 月時点で法令 9・ガイドライン 10・サービス 18 を含む 120 ノード以上、
355 関係を収録。架空利用者 3 ケースで支援仮説生成のデモを実装。

考察:
個別専門員の負荷分散とコミュニティ集合知化を両立する仕組みとして有効と考える。
今後は地域差対応・多言語化・他領域（高齢・児童）展開を検討する。

公開:
https://github.com/kazumasakawahara/welfare-graph （MIT + CC BY-SA 4.0）
```

## 7. メディア対応 Q&A 想定

**Q: なぜ無料で公開するのか？**
A: 相談支援は人の人生に関わる業務。情報格差で利用者の不利益が生じてはいけない。
コミュニティの共有資産として育てたい。

**Q: 実名や個人情報のリスクは？**
A: 仮名 ID（P-XXXX 等）の運用と、実名マップの暗号化（Meld Encrypt）、
書き込み時の自動検知 hook で多層防御。GitHub に上がるのは仮名情報のみ。

**Q: 法的・医学的助言になりませんか？**
A: 学習・参考目的に限定。個別判断は必ず担当者・医師・弁護士の合議で行う旨を
免責事項に明記。

**Q: 商用利用は？**
A: ライセンスで明示的に許可。事業所内利用も研修教材化も可能。
ただし改変版を再配布する場合はドキュメントは CC BY-SA 4.0 継承が必要。

**Q: 改正に追いつけますか？**
A: 自動検知の仕組みは内蔵していますが、最終的な情報の正確性はコミュニティの
継続的な貢献に依存します。1 人で全てを背負わないモデルです。
