## 導入・環境について

### Q. 本当に無料ですか？

**はい、完全無料** です。

- Obsidian 本体: 個人利用無料（商用利用は有料ライセンスが必要な場合あり）
- welfare-graph: MIT + CC BY-SA 4.0 のデュアルライセンスで**商用利用含め完全無料**
- プラグイン（Meld Encrypt 等）: すべて無料

個人の相談支援専門員・事業所での利用ともに、コストはかかりません。

### Q. インターネット接続は必須ですか？

**セットアップ時のみ必要** です。

- 初回: Obsidian ダウンロード・プラグインインストール・welfare-graph の ZIP ダウンロード
- 以降: **オフラインでも全機能が動作** します

訪問先・出張先でも問題なく使えます。

### Q. スマホ・タブレットでも使えますか？

Obsidian には iOS・Android 版もあります。ただし以下の制約あり:

- **ファイル同期が必要**: iCloud / Google Drive / Syncthing などで同期設定
- **表示は見られるが編集は困難**: 画面が小さいので入力作業には不向き
- **プラグインは一部のみ対応**: Meld Encrypt はモバイル版でも動作

**基本は PC で運用し、出先で閲覧のみモバイル** という使い方が現実的です。

### Q. Mac と Windows、どちらが使いやすい？

**どちらでも同じように使えます**。

好みや職場環境で選んでください。Obsidian はクロスプラットフォーム対応で、**保管庫（vault）を OS 間でコピーしても問題なく動きます**。

### Q. 他の相談員（同僚）と共有したい

以下の選択肢があります:

| 方法 | メリット | デメリット |
|---|---|---|
| **共有フォルダ経由**（事業所内 LAN） | 簡単 | 同時編集は競合リスク |
| **クラウド同期**（Dropbox/OneDrive） | 外出先でも同期 | 個人情報を外部に預ける |
| **Git + GitHub Private**（上級者） | 履歴管理完璧 | 技術ハードル高 |
| **個人運用のまま・zip でやり取り** | シンプル | リアルタイム性なし |

事業所の方針に合わせて選択してください。**特に個人情報保護の観点から、クラウド同期の利用は事業所の情報管理責任者と相談することを推奨**。

### Q. 既存の電子カルテや支援計画ソフトと連携できる？

直接の API 連携機能はありません。ただし:

- **エクスポート**: Obsidian の Markdown ファイルは単なるテキスト。他のソフトにコピペ可能
- **インポート**: PDF・Word 文書を貼り付けたり、要約を Markdown で記録可能

計画相談ソフトと welfare-graph は**併用運用**が現実的です:
- 計画相談ソフト: 公式な計画書・報酬請求
- welfare-graph: 思考整理・知識蓄積・根拠確認

## セキュリティ・プライバシー

### Q. 実名を書いたらどうなりますか？

**自動的にブロックされます**。

`.claude/hooks/realname_detect.py` という仕組みが、Claude Code 経由でファイルを書き込む際に実名パターン（漢字フルネーム・カタカナフルネーム・電話番号）を検出して書き込みを拒否します。

ただし:
- Obsidian で直接手打ちした場合はブロックされない（Obsidian 側の hook は別途必要）
- 定期的に `realname-check` スキルで監査することを推奨

### Q. 「実名らしき」と誤検知されたら？

「支援 方針」のような**漢字スペース漢字** の組み合わせが人名パターンに似ているため、誤検知されることがあります。

**対処法**: ノートの冒頭に以下を 1 行追加:

```
<!-- allow-realname -->
```

これで例外扱いとなり、書き込みが許可されます。

### Q. PC を紛失したらどうなる？

最悪のシナリオを想定した設計になっています:

1. **実名は暗号化ファイル（alias_map.md）にのみ**存在
2. Meld Encrypt は **AES-256** という強力な暗号を使用
3. パスワードが漏れない限り、実名の漏洩は起こらない

**推奨される対策**:

- **PC 本体も暗号化**（Mac: FileVault / Windows: BitLocker）を有効化
- **定期バックアップ** を暗号化された外付けドライブに保存
- **紛失時は即座に事業所管理者と情報管理責任者に報告**

### Q. alias_map.md のパスワードを定期変更した方がいい？

年 1 回程度の変更を推奨します:

1. `Cmd+P` → 「Decrypt note in-place」で復号
2. 内容を確認して保存（一時的にプレーンテキスト）
3. 「Encrypt note in-place」で新しいパスワードを設定

**パスワード管理アプリに記録**してから変更してください。

### Q. 事業所の監査で見せることができますか？

見せ方次第で可能です:

- **実名が不要な場合**: 仮名 ID のままで十分（ノート本文は仮名のみ）
- **実名が必要な場合**: 必要な時だけ `alias_map.md` を復号化して閲覧
- **監査終了後は必ず再暗号化**

## 日常の使い方

### Q. 1 日どれくらい時間をかけるのがベスト？

以下が目安です:

- **訪問・面談前の準備**: 1 ケース 5 分（該当ノートを開く）
- **訪問後の記録**: 1 件 10-15 分（エピソードノート作成）
- **月次まとめ**: 月 1 時間程度
- **知識ノート追加**: 気になった時に少しずつ（無理しない）

**完璧を目指さない** ことが継続のコツです。

### Q. どこから入力を始めるのがいい？

おすすめの順序:

1. **現在担当している利用者の中で、最も情報整理が必要な人 1 名** のノート
2. 次に、**困難ケース 1-2 名**
3. 慣れてきたら、**新規ケース** は必ず welfare-graph で管理開始
4. 余裕があれば、既存ケースを**徐々に移行**

全員を一度に入力しようとしないでください。

### Q. 過去の紙カルテを入力すべき？

**過去 1 年分程度** に絞ることを推奨:

- それ以前の情報は**要約のみ**記載
- 詳細な履歴が必要な時は紙カルテを参照
- welfare-graph の強みは「**これから**の支援計画の質向上」

過去の入力に時間を取られると本末転倒です。

### Q. 書くのが大変...

**ハードルを下げる工夫**:

1. **音声入力を使う**: Mac/Windows どちらも標準機能で音声→テキスト変換可能
2. **テンプレート活用**: `90_Meta/templates/person.md` をコピーして埋めていく
3. **箇条書きで構わない**: 美しい文章にこだわらない
4. **毎日少しずつ**: 訪問 1 件につき 5 分だけ記録

### Q. 書いた内容の検索方法は？

3 種類の検索があります:

| 検索 | ショートカット | 用途 |
|---|---|---|
| **クイックスイッチャー** | `Cmd+O` / `Ctrl+O` | ファイル名を打つと候補が出る |
| **全文検索** | `Cmd+Shift+F` / `Ctrl+Shift+F` | 本文のキーワード検索 |
| **タグ検索** | `#タグ名` | 特定のタグが付いたノート |

### Q. 印刷はできますか？

はい、Obsidian の印刷機能で PDF 化・紙印刷ができます:

- ノートを開いた状態で `Cmd+P` / `Ctrl+P`
- PDF として保存 または プリンターで印刷
- ただし **実名が含まれていないことを確認** してから印刷

## Obsidian の操作

### Q. ショートカットキー一覧

よく使うもの:

| 動作 | Mac | Windows |
|---|---|---|
| 新規ノート作成 | `Cmd+N` | `Ctrl+N` |
| ファイル検索 | `Cmd+O` | `Ctrl+O` |
| 全文検索 | `Cmd+Shift+F` | `Ctrl+Shift+F` |
| グラフビュー | `Cmd+G` | `Ctrl+G` |
| 戻る | `Cmd+Alt+←` | `Ctrl+Alt+←` |
| 進む | `Cmd+Alt+→` | `Ctrl+Alt+→` |
| 編集/閲覧切替 | `Cmd+E` | `Ctrl+E` |
| 保存 | `Cmd+S` | `Ctrl+S`（自動保存も有効） |
| コマンドパレット | `Cmd+P` | `Ctrl+P` |
| 設定 | `Cmd+,` | `Ctrl+,` |

### Q. ウィンドウを 2 つ並べたい

**Obsidian 画面内で分割** できます:

- ノートのタブを**右端に引き寄せる**と画面が 2 分割される
- または タブを**右クリック → 「右に分割」**

例: 法令を見ながら利用者ノートを書ける。

### Q. プレビューと編集を切り替えたい

- `Cmd+E` / `Ctrl+E` で切替
- 編集モード: Markdown 記法が見える
- プレビューモード: 整形された表示

**「ライブプレビュー」** モード（標準）なら両方の中間的な表示で便利です。

### Q. ダークモードにしたい

設定 → 外観 → ベーステーマ → 「ダーク」

目に優しく、夜の作業に最適です。層の色は自動で調整されます。

### Q. フォントを変えたい

設定 → 外観 → フォント → お好みで変更。
日本語フォントなら「游ゴシック」「ヒラギノ角ゴ」「Noto Sans CJK JP」あたりが読みやすいです。

## 複数人での運用

### Q. 事業所全体で 10 名で使いたい

運用モデル例:

**モデル A: 個人用途＋共有知識** (推奨・初期)
- 各人が自分の PC に個人用 vault（利用者情報）
- 知識層（60〜67 層）のみ Git で共有・定期更新
- 月 1 回の事例検討会で `30_Insights/` を持ち寄り

**モデル B: 中央リポジトリ**（上級・全員が技術的）
- 事業所内 NAS または Git サーバーで vault 共有
- 利用者担当ごとにフォルダ分け
- 実名マップは管理者のみ復号可能

**モデル C: ハイブリッド**
- 利用者情報: 各自ローカル
- 知識層・Insight: 事業所共通
- GitHub Private リポジトリで共有

### Q. 担当変更時の引き継ぎは？

1. **利用者ノートを前任者から後任者へ転送**（USB 等で）
2. **alias_map.md のパスワードを安全な方法で伝達**
3. 後任者は自分の PC に取り込み、自分のパスワードに変更可
4. できれば**口頭引き継ぎ**で、ノートに書いていない暗黙知も共有

### Q. 事業所を辞める時は？

1. **担当していた利用者情報** は事業所に引き継ぎ
2. **知識層（60-67）** は個人で継続利用可（ライセンス上問題なし）
3. **Insight（30_Insights）** は個人の学びとして持ち出し可

個人情報保護法の観点から、**退職時には必ず実名を含むファイルを削除**してください。

## トラブルシューティング

### Q. アプリが重くなった

原因の候補:

1. **ノート数が数千を超えた** → パフォーマンス設定で改善
   - 設定 → エディター → 「大きなノートの遅延レンダリング」を ON
2. **グラフビューが重い** → 表示するノード数を制限
   - グラフビュー設定 → フィルター → 特定フォルダのみ表示
3. **プラグインの干渉** → 1 つずつ無効化して原因を特定

### Q. リンクが大量に壊れた

連続して外字・特殊文字を含むファイル名で起きることがあります:

1. **ゴミ箱から復元** できないか確認
2. `vault_health_check.py`（上級編）で壊れリンクを一括検出
3. 手動で修正

予防策: **ファイル名は英数字＋基本的な日本語のみ** にする（絵文字・特殊記号を避ける）

### Q. 突然、新しい OS のバージョンで Obsidian が動かなくなった

1. Obsidian 公式 https://obsidian.md/download で**最新版**をダウンロードして上書きインストール
2. それでもダメなら公式フォーラム https://forum.obsidian.md で類似事例を検索

### Q. 「グラフに線が引かれない」「ノードだけ表示される」

Neo4j での話ではなく、Obsidian のグラフビューで発生する場合:

- 単一フォルダのみ表示している設定になっている可能性
- グラフ設定 → フィルター → 「未解決のリンクを非表示」の設定を確認

## 発展的な使い方

### Q. バックアップはどうすればいい？

**週 1〜月 1 回** の以下のいずれかを推奨:

**レベル 1: 手動コピー**（初心者向け）
- 外付け USB ドライブに `welfare-graph` フォルダ全体をコピー
- クラウドドライブ（個人用）に同期

**レベル 2: 自動バックアップ**
- Mac: Time Machine
- Windows: ファイル履歴
- 両 OS で対応: Syncthing 等

**レベル 3: Git**（上級者向け）
- 変更を細かく記録
- 任意の時点に戻せる
- GitHub Private リポジトリで自動バックアップ

### Q. 新しい知識ノートを追加したい

1. **適切な層** を選ぶ（例: 新しい法令なら `60_Laws/`）
2. テンプレートをコピー（`90_Meta/templates/law.md` 等）
3. frontmatter を埋める（`source_url`・`version`・`review_due` など必須）
4. 本文を記述
5. 関連する既存ノートから wikilink を張る

### Q. 定期的に健全性を確認したい

`90_Meta/scripts/vault_health_check.py` を実行すると、以下を自動チェック:

- 期限切れのレビュー（review_due 超過）
- 必須フィールド欠損
- 壊れた wikilink
- 孤児ノート（どこからも参照されていない）

実行には Python が必要です（上級者向け）。

### Q. GitHub で更新を受け取りたい

リポジトリを **Watch** すると、新しい知識追加・法令改定対応があった際に通知が届きます:

1. https://github.com/kazumasakawahara/welfare-graph を開く
2. 右上の **「Watch」** ボタンをクリック
3. **「All Activity」** または **「Releases only」** を選択

### Q. GitHub に書き込むにはアカウントが要る？

- **閲覧のみ**: アカウント不要
- **Issue 提出・コメント**: 無料アカウント必要
- **ダウンロード（ZIP）**: アカウント不要
- **Pull Request**: 無料アカウント必要
