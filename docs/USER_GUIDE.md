## 1. これは何か？

**welfare-graph** は、相談支援専門員のための「**障害福祉の知識ノート集**」です。

### できること

- **法令・ガイドライン・サービス・支援技法** がひとつにまとまっている
- **利用者さんごとのノート** を作って、関連情報を紐づけられる
- 画面上で **関係が見える**（例：「この人の状況なら、この法令と、このサービスと、この技法が関係する」）
- 書いたノートを **後から検索しやすい**
- 複数人で使う場合も、**実名が漏れないよう仕組みで守られている**

### できないこと・注意

- これは **参考資料・メモ帳の強化版** であって、法的・医学的判断をする道具ではありません
- 個別ケースの最終判断は、必ず担当の相談支援専門員・医師・弁護士等が行います
- 法令・制度は変わるので、最新情報は公式サイトで確認が必要です

### どんな環境で動くか

- **パソコン**: Windows または Mac（どちらでも可）
- **ネット接続**: セットアップ時のみ必要。日常利用はオフラインでも OK
- **費用**: **完全無料**

## 3. STEP 1: Obsidian をインストールする

**Obsidian（オブシディアン）** とは、文章とリンクで知識を整理するメモアプリです。無料。

### Mac の方

1. ブラウザで **https://obsidian.md/** を開く
2. 画面中央の **「Download」** ボタンをクリック
3. 「For macOS」 をクリック（Apple Silicon と Intel の選択が出る場合は、新しい Mac なら Apple Silicon・古い Mac なら Intel）
4. ダウンロードされた `.dmg` ファイルをダブルクリック
5. 出てきたウィンドウで、**Obsidian のアイコンを Applications フォルダにドラッグ**
6. Applications（アプリケーション）フォルダから **Obsidian を起動**

初回起動時に「開いてもよろしいですか？」というセキュリティ確認が出たら「開く」をクリック。

### Windows の方

1. ブラウザで **https://obsidian.md/** を開く
2. 画面中央の **「Download」** ボタンをクリック
3. 「For Windows」 をクリック
4. ダウンロードされた `.exe` ファイルをダブルクリック
5. インストーラが起動するので、**「次へ」を押し続ける**（特に設定変更は不要）
6. インストール完了後、**Obsidian を起動**

### うまくいかない場合

- 「開発元が未確認のため開けません」と出た場合（Mac）:
  - システム設定 → プライバシーとセキュリティ → 「このまま開く」
- ウイルス対策ソフトが警告を出した場合:
  - Obsidian は公式配布の安全なソフトです。「許可」「信頼する」を選択してください

## 4. STEP 2: welfare-graph をダウンロードする

ファイルを取得する方法は 2 つあります。**初めての方は 方法 A（ZIP ダウンロード）** をおすすめします。

### 方法 A: ZIP ダウンロード（推奨・簡単）

1. ブラウザで **https://github.com/kazumasakawahara/welfare-graph** を開く
2. 画面右上の緑色の **「Code」** ボタンをクリック
3. 出てきたメニューの一番下の **「Download ZIP」** をクリック
4. ダウンロードフォルダに `welfare-graph-main.zip` が保存される
5. **ZIP ファイルをダブルクリック** して展開（Windowsは右クリック→「すべて展開」）
6. 展開されたフォルダ **`welfare-graph-main`** を、わかりやすい場所に移動
   - **Mac のおすすめ場所**: `書類（Documents）` フォルダ の中
   - **Windows のおすすめ場所**: `ドキュメント` フォルダ の中
7. フォルダ名を **`welfare-graph`** に変更（`-main` を削除）してスッキリさせると、あとで見つけやすいです

### 方法 B: Git でクローン（上級者向け・更新が楽）

Git をすでに使ったことがある方のみ:
```bash
cd ~/Documents    # 好きな場所へ
git clone https://github.com/kazumasakawahara/welfare-graph.git
```

初心者の方は方法 A で進めてください。

### うまくいかない場合

- ZIP が展開できない → 別の展開ソフト（The Unarchiver 等）を使う
- どこに保存したか分からない → Finder（Mac）/ エクスプローラー（Windows）の「ダウンロード」フォルダを確認

## 5. STEP 3: Obsidian で開く

### 手順

1. Obsidian を起動する
2. 初回起動時は「**Create new vault**（新しい保管庫を作る）」と「**Open folder as vault**（フォルダを保管庫として開く）」の選択画面が出る
3. **「Open folder as vault」** をクリック
4. ファイル選択ダイアログで、先ほど保存した **`welfare-graph` フォルダ** を選択し、「開く」をクリック

**「保管庫（Vault）」** とは、Obsidian がひとまとめに扱うノート群の入れ物です。
`welfare-graph` フォルダ全体が 1 つの保管庫になります。

### 2 回目以降の起動

Obsidian は **前回開いた保管庫を覚えている** ので、2 回目以降は自動で welfare-graph が開きます。

別の保管庫に切り替えたい場合: 左下の保管庫アイコンから選択できます。

### 画面の見方

最初に見える画面の構成:

```
┌─────────────┬──────────────────────────┬─────────┐
│             │                          │         │
│  左サイド   │    ノートの本文         │ 右サイド│
│  バー       │    （中央のエリア）     │ バー    │
│             │                          │         │
│ フォルダ・  │                          │ バック  │
│ ファイル一覧│                          │ リンク等│
│             │                          │         │
└─────────────┴──────────────────────────┴─────────┘
```

左サイドバーに以下のフォルダが並んでいれば成功です:
- `00_MOC/`
- `10_People/`
- `20_Episodes/`
- `30_Insights/`
- `40_Stakeholders/`
- `50_Resilience/`
- `60_Laws/` 〜 `67_Orgs/`
- `90_Meta/`
- `raw/`
- その他 `README.md` など

### うまくいかない場合

- フォルダが展開されていない場合、各フォルダ名の左の三角マーク `▶` をクリック
- サイドバーが表示されていない場合、左上のアイコンで表示/非表示を切り替え

## 6. STEP 4: 色を付ける（見やすくする）

ノートの種類（法令・サービス・技法...）ごとに色分けすると、一目で区別しやすくなります。

### 手順

1. Obsidian 画面 **左下の歯車アイコン ⚙** をクリック（または `Cmd+,` / `Ctrl+,`）
2. 左側メニューから **「外観」** を選択
3. 画面を下にスクロールして **「CSS snippets（CSS スニペット）」** セクションを探す
4. `layer-colors` という項目が表示されているはず
5. **右側のトグルスイッチを ON**（青色）にする
6. 設定画面を閉じる（右上の × か、Esc キー）
7. 左サイドバーのフォルダ名が **色分けされた** ことを確認

正しく色分けされると:
- `60_Laws` が **赤**
- `64_Methods` が **緑**
- `66_Services` が **青**
- その他の層もそれぞれの色

### グラフビューにも色を付ける（より見やすく）

1. 左サイドバーで、グラフビューアイコン（丸と線がつながった図）をクリック
   または `Cmd+G` / `Ctrl+G`
2. 画面左上の **フィルターアイコン（漏斗マーク）** をクリック
3. **「カラーグループ（Color groups）」** を展開
4. 14 個のグループ（60_Laws, 61_Guidelines, ...）が表示されていれば成功

もし空欄の場合: [よくある困りごと](#12-よくある困りごと) の「グラフに色が付かない」を参照。

### うまくいかない場合

- `layer-colors` が項目に出ない → リフレッシュボタン（🔄）をクリック
- 色が付かない → Obsidian を一度終了して再起動
- 詳細な手順は [`90_Meta/SETUP_COLORS.md`](../90_Meta/SETUP_COLORS.md) を参照

## 7. STEP 5: Meld Encrypt プラグインを入れる

**実名リスト（次の STEP で作る）を暗号化するため** の重要なプラグインです。

### 手順

1. Obsidian 左下の **歯車アイコン ⚙** をクリック
2. 左側メニューから **「コミュニティプラグイン」** を選択
3. 初回は「**コミュニティプラグインを有効化**」というボタンが出るのでクリック
   - 「**制限モードを無効化する**」などの警告メッセージが出たら承諾
4. **「閲覧（Browse）」** ボタンをクリック
5. 検索欄に **`Meld Encrypt`** と入力
6. 検索結果から **Meld Encrypt** を選んでクリック
7. **「インストール」** ボタンをクリック（インストールに数秒）
8. インストール完了後、**「有効化」** ボタンをクリック
9. 設定画面を閉じる

### 動作確認

Obsidian 画面 左下の **コマンドパレット**（`Cmd+P` / `Ctrl+P`）で「encrypt」と入力して、関連コマンドが出てくれば成功です。

### うまくいかない場合

- 「コミュニティプラグインを有効化」ボタンが出ない → Obsidian を再起動
- 検索しても出ない → 別のキーワード（`encrypt`）で試す

## 8. STEP 6: 実名マップ（大切なファイル）を作る

相談支援業務では**実名を扱うことが法的・倫理的にリスク** です。
このシステムでは、ノート本文では **仮名 ID**（`P-0001` 等）だけを使い、
**実名との対応は暗号化した 1 つのファイル** にだけ記録します。

### 仮名 ID の体系

| 接頭辞 | 対象 |
|---|---|
| `P-XXXX` | 当事者本人（Person） |
| `F-XXXX` | 家族（Family） |
| `O-XXXX` | 事業所（Office） |
| `M-XXXX` | 医療機関（Medical） |
| `G-XXXX` | 成年後見人等（Guardian） |
| `S-XXXX` | 社会福祉協議会 |
| `C-XXXX` | 相談支援専門員（Consultant） |
| `N-XXXX` | その他（Neighbor・民生委員等） |

`XXXX` は 4 桁の数字（0001 から順番）。

### ファイル作成手順

1. 左サイドバーで `90_Meta` フォルダの中にある **`alias_map.md`** をクリック
   - もし存在しない場合は、90_Meta フォルダを右クリック → 「新規ノート」→ ファイル名を `alias_map` に変更
2. 以下のような内容を記入（**この例は架空名です**）:

```markdown
# 実名 ↔ 仮名 対応表

## 当事者（P-）
- P-0001: 田中太郎（1998年生・知的障害）
- P-0002: 佐藤次郎（1991年生・強度行動障害）

## 家族（F-）
- F-0001: 田中花子（P-0001の母・1960年生）
- F-0002: 田中一郎（P-0001の父・故）

## 事業所（O-）
- O-0001: 社会福祉法人〇〇 生活介護××
- O-0002: NPO法人△△ 共同生活援助□□

## 医療機関（M-）
- M-0001: ▽▽クリニック（精神科・主治医: 山田医師）

## 相談支援専門員（C-）
- C-0001: 自分（〇〇 相談支援事業所）
```

3. ここまで書いたら **重要な手順**: このファイルを暗号化します

### 暗号化手順（最重要）

1. `alias_map.md` のノート画面が開いている状態で
2. `Cmd+P` / `Ctrl+P` でコマンドパレットを開く
3. **「Meld Encrypt: Encrypt note in-place」** と入力して Enter
4. **パスワードを聞かれる** ので、**覚えやすい・自分だけが知っているパスワード** を入力
   - 例: `Tanaka-2026-consultation`（必ず自分だけの文字列にしてください）
5. 確認のためもう一度同じパスワードを入力
6. 一瞬でファイルが **暗号のような文字列に変わる** → 成功
7. **⚠ パスワードは絶対に忘れない**（パスワード管理アプリ・紙のメモ・金庫等に保管）

### 読みたい時は

1. `alias_map.md` を開くと暗号のような表示になっている
2. `Cmd+P` / `Ctrl+P` → **「Meld Encrypt: Decrypt note in-place」**
3. パスワードを入力 → 元の読める状態に戻る
4. **確認が終わったら必ず再暗号化**（同じコマンドで元に戻せる）

### うまくいかない場合

- **パスワードを忘れた** → **復元できません**。作り直しになります（仮名 ID は残せば、再入力で復元可能）
- 暗号化コマンドが出ない → Meld Encrypt プラグインが有効か再確認

### よくある質問

**Q. なぜこんなに面倒な仕組みが必要？**
A. 相談支援専門員の情報は **個人情報保護法上の要配慮個人情報** にあたります。万が一 USB 紛失・PC 盗難等が起きても、実名が特定されない形にしておくことで、**大きな情報漏洩事故を防ぐ** ことができます。

**Q. 同僚と共有したい場合は？**
A. 同じ事業所内で共有する場合、**共有パスワードを決めて** 同じファイルを複数端末で利用できます。ただし **パスワードの共有方法は慎重に**（対面で伝える・パスワード管理アプリ等）。

## 10. STEP 8: 最初の利用者ノートを作る

実際の業務で、1 人の利用者さんのノートを作ってみましょう。

### 事前準備

- 担当する利用者さんを 1 人選ぶ
- 次の仮名 ID を決める（例: 既存に P-0001〜P-0003 があるので `P-0004` から始める）
- `alias_map.md` を **復号化** して、P-0004 の実名を記入、**再暗号化**

### ノート作成手順

1. 左サイドバーの **`10_People`** フォルダを右クリック
2. **「新規ノート」** を選択
3. ファイル名に **`P-0004`** と入力して Enter
4. 空のノートが開く
5. 以下の骨組みをコピペ:

```yaml
---
type: person
id: P-0004
status: active
diagnosis: [知的障害（軽度）, 自閉スペクトラム症]
disability_cert: 療育手帳B
service_plan_id: SP-0004
primary_supporter: F-0004
cssclasses: [layer-person]
created: 2026-04-20
updated: 2026-04-20
tags: [person]
relations:
  - to: "[[63_Disorders/知的障害]]"
    type: has-characteristic
    weight: 0.8
  - to: "[[60_Laws/障害者総合支援法]]"
    type: applies-to
    weight: 1.0
  - to: "[[61_Guidelines/意思決定支援ガイドライン]]"
    type: compliance-required
    weight: 0.9
---
```

（↑ `diagnosis` 等は P-0004 さんの実態に合わせて書き換え）

6. 続けて本文を書きます:

```markdown
# P-0004

## 基本情報
- 年齢 / 性別: 25歳 / 女性
- 居住形態: 実家（両親と同居）
- 収入: 障害基礎年金2級 + 就労継続B型工賃

## 強み（Strengths）
- ここに得意なこと・好きなことを書く

## 禁忌 / 苦手（Contraindications）
- ここに苦手なこと・避けるべき対応を書く
```

## 12. よくある困りごと

### Q1. Obsidian が起動しない

- **Mac**: システム設定 → プライバシーとセキュリティ → 「このまま開く」で許可
- **Windows**: ウイルス対策ソフトが干渉している可能性。例外リストに Obsidian を追加
- それでもダメ → 一度アンインストールして最新版を再インストール

### Q2. ファイルが見つからない

- 左サイドバーのフォルダ名の左の三角 `▶` をクリックして展開
- 検索: `Cmd+O` / `Ctrl+O` でファイル名検索
- 全文検索: `Cmd+Shift+F` / `Ctrl+Shift+F`

### Q3. 色が付かない

1. 設定 → 外観 → CSS snippets で `layer-colors` が ON になっているか確認
2. リフレッシュボタン 🔄 をクリック
3. Obsidian を終了して再起動

### Q4. グラフに色が付かない

`graph.json` が Obsidian により上書きされている可能性:

1. **Obsidian を完全に終了**（Cmd+Q / ウィンドウ閉じるだけではダメ）
2. 30 秒待つ
3. Obsidian を再起動
4. グラフビュー → カラーグループ → 14 個の項目が表示される

それでも空の場合は **手動で設定可能** です。カラーグループ → 「新しいグループ」ボタン → 検索欄に `path:60_Laws/` → 色を選択（赤） → 繰り返し 14 層分追加。

詳細は [`90_Meta/SETUP_COLORS.md`](../90_Meta/SETUP_COLORS.md)。

### Q5. 「実名らしき文字列を検出しました」と出てブロックされる

`.claude/hooks/realname_detect.py` が実名らしき文字列を検出したサイン。ただし **誤検知もあります**（例: 「支援 方針」のような漢字スペース漢字の組み合わせ）。

**対処法**:

- **本当に実名** が入ってしまっている場合: 仮名 ID に置き換える
- **誤検知** の場合: ノートの冒頭（`---` の直前）に以下を追加:
  ```
  <!-- allow-realname -->
  ```

### Q6. パスワードを忘れてしまった

**Meld Encrypt で暗号化したファイルは、パスワードなしでは復元できません**。セキュリティ上の仕様です。

- `alias_map.md` のパスワードを忘れた場合: ファイルを削除して、仮名 ID と実名の対応を書き直すしかありません。ノート本文は仮名 ID だけなので、業務は継続できます
- 今後の対策: **パスワード管理アプリ**（1Password / Bitwarden 等）に保存しておくのが最も安全

### Q7. ノートを間違えて消してしまった

Obsidian には標準で **ゴミ箱機能** があります:
- Mac: `.trash/` フォルダ（隠しフォルダ）または OS のゴミ箱
- Windows: OS のゴミ箱

30 日以内なら復元可能（OS の仕様による）。

**定期的なバックアップ** をおすすめします（後述「発展的な使い方」参照）。

### Q8. 「このリンクは解決されません」と出る

`[[あああ]]` のように書いてリンク先が存在しない場合、Obsidian は「未解決のリンク」として表示します。

**対処法**:
- タイポの場合: 正しく書き直す
- まだノートを作っていない場合: リンクをクリックすると新規ノートが作られる

### Q9. 書いたノートに間違いがあった

通常の文字編集と同じ。直接書き換えて `Cmd+S` で保存。

**履歴を残したい場合**:
- Obsidian 設定 → 「ファイル復元」で過去の版を確認可能（有効化が必要）
- または Git で管理（上級者向け）

### Q10. 新しいパソコンに移したい

1. 現在の `welfare-graph` フォルダ **全体** をコピー（外付けドライブ・クラウドドライブ経由）
2. 新しい PC で Obsidian を起動 → 「Open folder as vault」 → コピーしたフォルダを選択
3. `layer-colors` の CSS snippet を有効化・Meld Encrypt プラグインを再インストール

## 13. raw 資料の取り込み（外部資料を wiki 化する）

法令の改正・新ガイドライン・判例・Wikipedia 記事など、**外部から入手した資料を welfare-graph に取り込む** 方法です。

### 全体の流れ

```
[資料を入手]
  ↓
[raw/ の適切なサブフォルダに配置]
  ↓
[Claude Code で取り込み（または手作業）]
  ↓
[適切な層（60_Laws / 30_Insights 等）に wiki ページ生成]
  ↓
[既存ページとの relations を繋ぐ]
```

### raw/ サブフォルダの使い分け

| 資料の種類 | raw/ の置き場所 | 取り込み先 |
|---|---|---|
| 法令の原文 PDF | `raw/laws/` | `60_Laws/` |
| 厚労省ガイドライン PDF | `raw/guidelines/` | `61_Guidelines/` |
| サービス報酬告示 | `raw/services/` | `66_Services/` |
| 判例・事例・Wikipedia 記事 | `raw/insights/` | `30_Insights/` |
| 学会報告書 | `raw/reports/` | `30_Insights/` または `90_Meta/` |

### 方法 A: Claude Code で自動取り込み（推奨）

#### 手順

1. **資料を取得して raw/ に配置**

   例 1: 厚労省サイトから障害福祉サービス等報酬改定のPDFをダウンロードし、`raw/services/` に配置
   ```
   raw/services/障害福祉サービス費用算定基準_令和9年度改定.pdf
   ```

   例 2: Wikipedia の事件記事をコピペして MD ファイルに保存
   ```
   raw/insights/2024_旧優生保護法違憲判決.md
   ```

2. **Claude Code を起動**して、welfare-graph のフォルダで以下を実行:

   ```
   raw/services/障害福祉サービス費用算定基準_令和9年度改定.pdf を 66_Services/ に取り込んで
   ```

   または:
   ```
   raw/insights/2024_旧優生保護法違憲判決.md を wiki に取り込んでください
   ```

3. **Claude が自動で実行**:
   - 資料を読み込み
   - 適切な層・ファイル名を判定（既存ページと衝突しないよう）
   - frontmatter を生成（必須フィールドを埋める）
   - 本文を要約・構造化（A4 1-2 ページ程度）
   - 既存ページとの relations を提案（3-10 個）
   - 実名混入チェック（公人以外は役割表現に置換）

4. **承認確認** が表示されるので、内容を確認して承認

5. **完了報告** を確認:
   ```
   ✅ 取り込み完了

   新規ページ: 30_Insights/2024_旧優生保護法違憲判決.md
   relations: 5 個
   出典: raw/insights/2024_旧優生保護法違憲判決.md（原本保管）
   ```

#### 実例: 七生養護学校事件の取り込み

このリポジトリには実際に取り込んだ例があります:

- **元資料**: `raw/insights/2003_七生養護学校事件.md`（Wikipedia 記事ベース）
- **取り込み後**: `30_Insights/七生養護学校事件_性教育とバックラッシュ.md`

両方を見比べると、**「raw の生原稿」→「相談支援の文脈で構造化された wiki ページ」** の変換が確認できます。

### 方法 B: 手作業で取り込み（Claude を使わない場合）

Claude Code を使わない・使えない場合の手順:

1. **raw/ の資料を読む**（Obsidian でも別のビューアでも OK）

2. **取り込み先層のテンプレート** をコピー:
   - 法令なら `90_Meta/templates/law.md`
   - ガイドラインなら `90_Meta/templates/guideline.md`
   - 知見なら `90_Meta/templates/insight.md`
   - 各種テンプレートあり

3. **新しいノートを作成**:
   - 取り込み先フォルダ（例: `60_Laws/`）を右クリック → 新規ノート
   - ファイル名を決める（例: `障害者権利条約.md`）

4. **テンプレートを貼り付け** て、frontmatter の必須項目を埋める:
   - `source_url`: 公式 URL
   - `version`: 版名
   - `effective_date`: 施行日
   - `review_due`: 次回見直し予定日
   - `issuer`: 発行元

5. **本文を書く**:
   - 概要（1-3 段落）
   - 主要内容（資料の構造に沿って）
   - 計画相談業務との関係
   - 出典

6. **relations を追加**: 関連する既存ページに wikilink を張る

7. **保存** して完了

### 取り込み時の注意

#### 著作権配慮

- **原文 PDF は git に含めない**（自動的に `.gitignore` で除外）
- 公開リポジトリには **要約と出典 URL のみ** を載せる
- 原文の引用は **数行〜数段落程度** に留める（著作権法 32 条適法引用の範囲）

#### 実名の取扱い

| 種類 | 取扱い |
|---|---|
| 公人（議員・首長等の公式発言） | 引用 OK（記事に `<!-- allow-realname -->` を付与） |
| 当事者（生徒・利用者・職員） | **役割表現に置換**（「校長 X」→「元校長」） |
| 判決文の当事者名 | 基本的に役割表現で記述 |

#### 一次情報の改変禁止

- **raw/ の元ファイルは絶対に編集しない**（原本として保管）
- 要約は wiki ページ側で行い、`source` フィールドで原本リンク

### より詳しいスキル仕様

`raw-to-wiki` スキルの完全仕様は [`.claude/skills/raw-to-wiki/SKILL.md`](../.claude/skills/raw-to-wiki/SKILL.md) 参照。

## 15. （任意・上級編）Neo4j で高度な分析

Neo4j（ネオフォージェイ）という **グラフデータベース** と連携すると、複雑な問い合わせが可能になります。

**注意**: これもオプションです。地域の傾向分析・複数利用者比較等、研究的な用途向け。

### 例: Neo4j でできること

- 「3 名の利用者に共通して関連する法令は？」
- 「ある障害特性から 3 ホップ以内で到達可能なサービス・機関・技法は？」
- 「参照頻度が高い技法 Top 10」

### 必要なもの

- ターミナル操作への抵抗感がないこと

4. Neo4j コンテナを起動（README 参照）
5. `python3 90_Meta/scripts/sync_to_neo4j.py` で同期
6. ブラウザで http://localhost:7474 を開いて Cypher クエリ実行

詳細設計は [`90_Meta/neo4j-integration-design.md`](../90_Meta/neo4j-integration-design.md) 参照。

## 16. 最低限守ってほしいこと

相談支援業務は **人の人生に関わる仕事** です。以下は必ず守ってください。

### 🔴 絶対にやってはいけない

1. **実名をノート本文に書かない**
   - frontmatter の `id:` 欄以外は、すべて仮名 ID（`P-0001` 等）
   - 実名と仮名の対応は `alias_map.md` のみ（Meld Encrypt で暗号化）

2. **パスワードを平文で保存しない**
   - `.env` ファイル（今回作成しませんが、上級編で使用）は必ず `.gitignore` で除外
   - `alias_map.md` のパスワードを付箋・メモ帳・Slack 等に書かない

3. **法令・サービス情報を鵜呑みにしない**
   - このシステムの内容は「執筆時点」の情報
   - **最新情報は必ず e-Gov・厚生労働省・自治体サイトで確認**
   - 特に**3 年ごとの報酬改定**（次回 2027 年度）で大きく変わる

4. **本人の意思を飛び越えた支援仮説を立てない**
   - AI が生成する仮説はあくまで **候補**
   - 最終判断は必ず **本人・家族・関係者の合議**
   - 意思決定支援ガイドラインに沿う

### 🟡 推奨すること

- **定期バックアップ**: 月に 1 回は USB / 外付けドライブ / クラウドにコピー
- **パスワード管理アプリ** の使用（1Password / Bitwarden 等）
- **月次の健全性チェック**: `vault_health_check.py` スクリプトを定期実行（上級編）
- **事業所内での情報共有ルール** の明文化

### 🟢 大切にしてほしい姿勢

- **タブー視しない**: 性・虐待・成年後見などの話題もしっかり記録
- **本人中心**: 支援者の都合ではなく、本人の願いを起点に
- **段階的に**: 一度に全部完璧にしようとしない
- **継続**: 3 年続けると、地域の支援知識の共有資産になります
